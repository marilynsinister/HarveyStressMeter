# 🔍 Отладка Social стресса

## Пошаговое тестирование

### Шаг 1: Получение дебаффа Social
1. Найдите NPC с дружбой **< 3 сердца** (например, Клинт, Пэм)
2. Поговорите с ним (30% шанс получения дебаффа)
3. **Ожидаемый результат**: 
   - Дебафф `buffStressSocial` применен
   - Топик `topicStressSocial` добавлен
   - В логе: `[Social Stress] Триггер активирован при разговоре с X (дружба: Y/750)`

### Шаг 2: Разговор с Харви
1. Найдите Харви и поговорите с ним
2. Должен появиться диалог с топиком `topicStressSocial`
3. В конце диалога должен установиться топик `topicStressTreatmentSocialStarted`
4. **Проверьте в логе**:
   ```
   [Диалог] Начался разговор с Харви. Текущие топики: topicStressSocial
   [Диалог] Дебафф Social активен: True
   ```

### Шаг 3: Начало лечения (автоматически через 1 секунду)
**Проверьте в логе**:
```
[Лечение] ═══ Найден топик начала лечения: topicStressTreatmentSocialStarted → Социальный дискомфорт ═══
[Лечение] Текущее состояние ПЕРЕД началом:
[Лечение]   - HasBuff(buffStressSocial): True
[Лечение]   - В ActiveLockedDebuffs: False
[Лечение]   - Всего разговоров сегодня: X
[Лечение] ▶ Вызов StartTreatment для Социальный дискомфорт...
[StartTreatment] Квест HarveyMod_SocialRecovery добавлен. Проверка: HasQuest=True
[QuestService.AddQuest] 'HarveyMod_SocialRecovery': было=False, стало=True, квестов N→N+1
[StartTreatment] ✅ Дебафф buffStressSocial добавлен в ActiveLockedDebuffs с квестом HarveyMod_SocialRecovery
[StartTreatment] Состояние после старта: ActiveLockedDebuffs.Count=1, HasBuff=True, HasQuest=True
```

**ЕСЛИ В ЛОГЕ**:
```
[QuestService.AddQuest] ⚠️ КВЕСТ НЕ ДОБАВЛЕН!
```
**ПРОБЛЕМА**: Квест `HarveyMod_SocialRecovery` не существует в `Data/Quests` или имеет ошибку

**ЕСЛИ В ЛОГЕ**:
```
[SyncQuestsAndBuffs] ⚠️ УДАЛЕНИЕ сиротского дебаффа buffStressSocial: квест HarveyMod_SocialRecovery не найден в журнале
```
**ПРОБЛЕМА**: Квест добавляется, но сразу исчезает из журнала

### Шаг 4: Проверка прогресса квеста
1. Поговорите с любым NPC (не Харви)
2. **Проверьте в логе**:
   ```
   [Диалог] Завершен разговор с X. Всего разговоров сегодня: N
   [Social Quest] Разговор #1 после получения квеста (база: 0, всего: N)
   ```

## Команды для отладки

### Полная диагностика системы
```
hs.debug
```

Покажет:
- Активные топики (Stress/Treatment)
- Активные баффы стресса (с замками)
- Залоченные дебаффы (квесты)
- Состояния дебаффов (StressBuffStates)
- Прогресс лечения

### Очистка всех состояний
```
hs.clear
```

### Просмотр всех состояний дебаффов
```
hs.states
```

## Что проверить при проблеме

### 1. Квест не добавляется
- Откройте `Data/Quests` в Content Patcher
- Проверьте наличие `HarveyMod_SocialRecovery`
- Проверьте формат квеста (должен быть тип `Basic`)

### 2. Дебафф пропадает сразу
Проверьте лог на наличие:
- `[SyncQuestsAndBuffs] ⚠️ УДАЛЕНИЕ сиротского дебаффа`
- Если есть - квест не добавляется в журнал

### 3. Счетчик разговоров не работает
Проверьте:
- Харви НЕ добавляется в `TalkedNpcsToday`
- После разговора с NPC появляется лог `[Social Quest] Разговор #N`
- `SocialTalksAfterQuest` увеличивается

### 4. Квест не завершается
Проверьте условия в логе:
- `conversationsAfterQuest >= 3 && SecondsNearHarvey >= 60` (первый вариант)
- `conversationsAfterQuest >= 5` (второй вариант)

## Критические точки для проверки

1. **Топик от Харви устанавливается?**
   - Диалог Харви должен содержать `#$t topicStressTreatmentSocialStarted 0`

2. **Квест существует в Data/Quests?**
   - `HarveyMod_SocialRecovery` должен быть в файлах квестов

3. **Дебафф восстанавливается каждую секунду?**
   - `EnsureLockedBuffsPersist()` должен применять дебафф, если он пропал

4. **SyncQuestsAndBuffs не удаляет дебафф?**
   - Квест должен быть в журнале, чтобы дебафф не удалился
